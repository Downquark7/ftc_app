54   /** 
55    * This OpMode illustrates the basics of using the Vuforia localizer to determine 
56    * positioning and orientation of robot on the FTC field. 
57    * The code is structured as a LinearOpMode 
58    * <p> 
59    * Vuforia uses the phone's camera to inspect it's surroundings, and attempt to locate target images. 
60    * <p> 
61    * When images are located, Vuforia is able to determine the position and orientation of the 
62    * image relative to the camera.  This sample code than combines that information with a 
63    * knowledge of where the target images are on the field, to determine the location of the camera. 
64    * <p> 
65    * This example assumes a "diamond" field configuration where the red and blue alliance stations 
66    * are adjacent on the corner of the field furthest from the audience. 
67    * From the Audience perspective, the Red driver station is on the right. 
68    * The two vision target are located on the two walls closest to the audience, facing in. 
69    * The Stones are on the RED side of the field, and the Chips are on the Blue side. 
70    * <p> 
71    * A final calculation then uses the location of the camera on the robot to determine the 
72    * robot's location and orientation on the field. 
73    * 
74    * @see VuforiaLocalizer 
75    * @see VuforiaTrackableDefaultListener 
76    * see  ftc_app/doc/tutorial/FTC_FieldCoordinateSystemDefinition.pdf 
77    * <p> 
78    * Use Android Studio to Copy this Class, and Paste it into your team's code folder with a new name. 
79    * Remove or comment out the @Disabled line to add this opmode to the Driver Station OpMode list. 
80    * <p> 
81    * IMPORTANT: In order to use this OpMode, you need to obtain your own Vuforia license key as 
82    * is explained below. 
83    */

183          /** 
184           * In order for localization to work, we need to tell the system where each target we 
185           * wish to use for navigation resides on the field, and we need to specify where on the robot 
186           * the phone resides. These specifications are in the form of <em>transformation matrices.</em> 
187           * Transformation matrices are a central, important concept in the math here involved in localization. 
188           * See <a href="https://en.wikipedia.org/wiki/Transformation_matrix">Transformation Matrix</a> 
189           * for detailed information. Commonly, you'll encounter transformation matrices as instances 
190           * of the {@link OpenGLMatrix} class. 
191           * 
192           * For the most part, you don't need to understand the details of the math of how transformation 
193           * matrices work inside (as fascinating as that is, truly). Just remember these key points: 
194           * <ol> 
195           * 
196           *     <li>You can put two transformations together to produce a third that combines the effect of 
197           *     both of them. If, for example, you have a rotation transform R and a translation transform T, 
198           *     then the combined transformation matrix RT which does the rotation first and then the translation 
199           *     is given by {@code RT = T.multiplied(R)}. That is, the transforms are multiplied in the 
200           *     <em>reverse</em> of the chronological order in which they applied.</li> 
201           * 
202           *     <li>A common way to create useful transforms is to use methods in the {@link OpenGLMatrix} 
203           *     class and the Orientation class. See, for example, {@link OpenGLMatrix#translation(float, 
204           *     float, float)}, {@link OpenGLMatrix#rotation(AngleUnit, float, float, float, float)}, and 
205           *     {@link Orientation#getRotationMatrix(AxesReference, AxesOrder, AngleUnit, float, float, float)}. 
206           *     Related methods in {@link OpenGLMatrix}, such as {@link OpenGLMatrix#rotated(AngleUnit, 
207           *     float, float, float, float)}, are syntactic shorthands for creating a new transform and 
208           *     then immediately multiplying the receiver by it, which can be convenient at times.</li> 
209           * 
210           *     <li>If you want to break open the black box of a transformation matrix to understand 
211           *     what it's doing inside, use {@link MatrixF#getTranslation()} to fetch how much the 
212           *     transform will move you in x, y, and z, and use {@link Orientation#getOrientation(MatrixF, 
213           *     AxesReference, AxesOrder, AngleUnit)} to determine the rotational motion that the transform 
214           *     will impart. See {@link #format(OpenGLMatrix)} below for an example.</li> 
215           * 
216           * </ol> 
217           * 
218           * This example places the "stones" image on the perimeter wall to the Left 
219           *  of the Red Driver station wall.  Similar to the Red Beacon Location on the Res-Q 
220           * 
221           * This example places the "chips" image on the perimeter wall to the Right 
222           *  of the Blue Driver station.  Similar to the Blue Beacon Location on the Res-Q 
223           * 
224           * See the doc folder of this project for a description of the field Axis conventions. 
225           * 
226           * Initially the target is conceptually lying at the origin of the field's coordinate system 
227           * (the center of the field), facing up. 
228           * 
229           * In this configuration, the target's coordinate system aligns with that of the field. 
230           * 
231           * In a real situation we'd also account for the vertical (Z) offset of the target, 
232           * but for simplicity, we ignore that here; for a real robot, you'll want to fix that. 
233           *

235           * - First we rotate it 90 around the field's X axis to flip it upright 
236           * - Then we rotate it  90 around the field's Z access to face it away from the audience. 
237           * - Finally, we translate it back along the X axis towards the red audience wall.

361      /*** 
362       * code sets variables lastLocation to a matrix and locationUpdated to true or false depending on if the camera sees a location 
363       */ 
364      public OpenGLMatrix getTrackingData() { 
365          locationUpdated = false; 
366          for (VuforiaTrackable trackable : allTrackables) { 
367              /** 
368               * getUpdatedRobotLocation() will return null if no new information is available since 
369               * the last time that call was made, or if the trackable is not currently visible. 
370               * getRobotLocation() will return null if the trackable is not currently visible. 
371               */ 
372              telemetry.addData(trackable.getName(), ((VuforiaTrackableDefaultListener) trackable.getListener()).isVisible() ? "Visible" : "Not Visible");    // 
373   
374              OpenGLMatrix robotLocationTransform = ((VuforiaTrackableDefaultListener) trackable.getListener()).getUpdatedRobotLocation(); 
375              if (robotLocationTransform != null) { 
376                  lastLocation = robotLocationTransform; 
377                  locationUpdated = true; 
378              } 
379          } 
380          /** 
381           * Provide feedback as to where the robot was last located (if we know). 
382           */ 
383          if (lastLocation != null) { 
384              //  RobotLog.vv(TAG, "robot=%s", format(lastLocation)); 
385              telemetry.addData("Pos", format(lastLocation)); 
386          } else { 
387              telemetry.addData("Pos", "Unknown"); 
388          } 
389          telemetry.update(); 
390          return lastLocation; 
391      } 
392   
393      public int getDegreeOffset() { 
394          if (locationUpdated) { 
395              this.data = getTrackingData(); 
396              this.data.toVector(); 
397          } 
398          return 0; 
399      }

401      /** 
402       * A simple utility that extracts positioning information from a transformation matrix 
403       * and formats it in a form palatable to a human being. 
404       */ 
405      String format(OpenGLMatrix transformationMatrix) { 
406          return transformationMatrix.formatAsTransform(); 
407      }
